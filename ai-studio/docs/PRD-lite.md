## 1. North Star (아하 루프 30분)

### 한 줄 선언
> "우리는 '동기부여'가 아니라 '구조'로 행동을 바꾼다."

### 아하 모먼트 정의
사용자가 30분 후 느끼는 감정:
- "나 오늘 했다."
- "내가 나에게 한 약속을 지켰다."
- "흔적이 남아서 존재가 증명됐다."

### 30분 아하 사이클 (제품의 심장)
```
태스크 등록(1줄) → 30분 진행 → 완료(버튼) → 증명 생성 → (선택) 스토리 저장
```

### 핵심 질문 (모든 기능의 심사 기준)
> "이 기능이 30분 아하 경험을 더 빠르고 더 확실하게 만드는가?"

### North Star Metric (NSM)
**D7 Active Task Completion Rate**
- 설치 후 7일 동안, 최소 1회 이상 '30분 완료+증명 생성'에 도달한 비율

### KPI 5개 (MVP)
| KPI | 정의 |
|-----|------|
| TTV | 설치 후 60초 내 '첫 시작' 비율 |
| 30분 완료율 | 시작 → 완료 버튼 → 증명 생성 |
| 증명 생성률 | 완료 → Proof 카드 생성 |
| 스토리 저장률 | 템플릿 이미지 Export 성공 |
| D7 리텐션 | 7일 내 1회 이상 완료+증명 |

## 4. v0.1 핵심 원칙

### Standalone-First
- 서버 없이 완전 로컬 동작
- IndexedDB 기반 저장
- Export(내보내기)로 백업

### 1분 이탈 방지
- 1화면 온보딩
- 목표 1줄 → 30분 시작 (1행동)
- 옵션은 접어두기

### Feature Tag 규칙
- `[LOCAL]` : 네이티브로 옮겨도 동일하게 가능
- `[SERVER-PROTO]` : 프로토타입 편의용 (나중에 교체/삭제)
- `[SERVER-BM]` : 비즈니스 모델 핵심 (백업/동기화)

> v0.1은 모든 기능이 `[LOCAL]`

## 5. 완료 정의 (DoD)

**"타이머 끝"이 아니라 "완료 버튼 + 증명 생성"**

```
시작 → 타이머 실행 → 완료 버튼 클릭 → Proof 자동 생성 → 아하!
```

- 타이머 자동 종료가 아닌, 사용자의 **완료 버튼** 클릭이 기준
- 완료 시 Proof 카드 자동 생성
- 0분 완료도 허용 ("짧아도 괜찮아요")
